優化器遊樂場

Optimizer Playground

訓練神經網路的核心:如何在損失地形上找到最低點。 下面可以看不同優化器怎麼走,哪個更快、更穩。

優化器做什麼?
根據梯度決定怎麼更新參數,目標是讓 loss 越來越小。就像在一片山谷中找最低點。
為什麼 SGD 不夠?
Vanilla SGD 對所有參數用同一個 learning rate,遇到狹長峽谷會劇烈震盪,遇到鞍點則會卡住。
地形比喻
想像你是一顆球在山坡上滾。SGD 是無慣性的,每步只看當下坡度;Momentum 像有質量,能衝過小丘;Adam 像有 GPS,知道該快還是慢。
Adaptive vs Non-adaptive
Non-adaptive(SGD、Momentum）: 固定 lr。Adaptive（Adam、AdaGrad）: 根據過去梯度自動調整每個參數的 lr。

SGD — 隨機梯度下降

Stochastic Gradient Descent
θ = θ − η ∇L(θ)

最簡單的優化器:沿梯度方向走一步,步長 = learning rate η。

問題 1: 震盪 — 在狹長峽谷中,梯度方向與最低點方向垂直,會來回擺盪。
問題 2: 學習率敏感 — 太大會發散,太小會卡在局部最小值或鞍點。
問題 3: 鞍點 — 在梯度接近零的地方幾乎不動。高維空間中鞍點遠比局部最小值多。

SGD + Momentum — 帶慣性的下降

Momentum accelerates convergence
vt = μ · vt−1 − η ∇L(θ)
θ = θ + vt

v 像速度,μ（通常 0.9）讓球「記得之前的方向」。連續朝同一方向的梯度會累積速度,垂直方向的振盪則被抵消。

比喻: 一顆有質量的球在斜面上滾,即使小幅上坡也不會馬上停,而是靠慣性衝過去。
效果: 在峽谷中減少震盪,在平坦處加速通過。

Nesterov Accelerated Gradient

Look-ahead momentum
vt = μ · vt−1 − η ∇L(θ + μ · vt−1)
θ = θ + vt

標準 Momentum 先算梯度再動;NAG 先跳到預測位置再算梯度。 像一個「先看再跳」的智慧球 — 如果前面是上坡,它會提前煞車。

差異: 當快接近最低點時,NAG 能更早減速,避免衝過頭。
實務: 在 PyTorch 中只需 SGD(params, lr=0.01, momentum=0.9, nesterov=True)

Adam — 自適應矩估計

Adaptive Moment Estimation
mt = β₁ · mt−1 + (1 − β₁) · gt
vt = β₂ · vt−1 + (1 − β₂) · gt²
m̂t = mt / (1 − β₁t)
v̂t = vt / (1 − β₂t)
θ = θ − η · m̂t / (√v̂t + ε)
mt (第一矩): 梯度的指數移動平均 → 像 Momentum,記住方向。
vt (第二矩): 梯度平方的指數移動平均 → 記住幅度/方差。
Bias Correction: 初期 m 和 v 都偏向 0（因為初始值是 0），除以 (1 − β^t) 修正。
更新規則: 方差大的維度步長小(不確定時走慢),方差小的維度步長大(確定時走快)。

預設超參數: β₁ = 0.9, β₂ = 0.999, ε = 1e-8。幾乎所有 DL 任務的第一選擇。

PyTorch:
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3) for epoch in range(epochs): loss = criterion(model(x), y) optimizer.zero_grad() loss.backward() optimizer.step()

AdamW — 解耦權重衰減

Decoupled Weight Decay Regularization
θ = θ − η · (m̂t / (√v̂t + ε) + λ · θ)

Adam + L2 regularization 有個問題: L2 項會被 adaptive learning rate 縮放,導致大梯度參數的正則化反而弱。 AdamW 把 weight decay 從梯度更新中解耦出來,直接對權重做衰減,效果更穩定。

Adam + L2: 在梯度 g 中加入 λθ,然後 adaptive 縮放。正則化強度不均勻。
AdamW: 先做 Adam 更新,再額外減去 η·λ·θ。正則化不受 adaptive 影響。
實務: 現在大多數模型(BERT、GPT、ViT)訓練都用 AdamW。
PyTorch:
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=0.01)

實務建議
不確定用什麼的時候:
• CV 任務（圖像分類、物件偵測）→ SGD + Momentum + CosineAnnealing,已有幾十年的 recipe 最穩。
• NLP / Transformer / 新架構 → AdamW + Warmup + Cosine decay,收斂快、不需太多調參。
• 如果有預算調 hyperparameter → SGD 能找到更好的泛化,但需要更多 epochs。
