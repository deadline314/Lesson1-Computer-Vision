Data Augmentation Playground

Data Augmentation 是訓練時對圖片做隨機變換,讓模型學到「不變性」。 只在訓練時使用,不增加推論成本,卻能大幅提升泛化能力。

為什麼需要增強?
更多樣化的資料 = 更好的泛化。避免模型只記住訓練集的特定模式(過擬合)。
只在訓練時!
推論(inference)時不做增強,所以完全不增加部署成本。是免費的午餐。
三大類別
幾何變換(翻轉、旋轉、裁切)、光度變換(色彩、亮度)、進階合成(Mosaic、Mixup)。
核心哲學
教模型對「不相關的變化」保持不變性:左右翻轉不該改變預測,亮度變化不該影響偵測。

進階技術 Advanced Techniques

AutoAugment — 自動學習增強策略

Google 2018 年提出,用 Reinforcement Learning 搜尋最佳增強組合。在 ImageNet 上搜索出 25 個子策略,每個包含兩個增強操作。缺點:搜索成本極高(數千 GPU hours)。

Policy = { (op1, prob1, mag1), (op2, prob2, mag2) } × 25 sub-policies
RandAugment — 簡化版隨機增強

只需兩個超參數:N(操作數量)和 M(強度)。從 14 種操作中隨機挑 N 個,每個用相同強度 M。效果接近 AutoAugment,但搜索成本趨近零。

RandAugment(N=2, M=9): 隨機挑 2 個操作,強度 9/30
Albumentations 函式庫

Python 最流行的圖片增強庫,比 torchvision.transforms 快數倍。支援 bounding box 和 segmentation mask 的同步變換,是 YOLO 訓練的標配。

import albumentations as A
transform = A.Compose([
    A.HorizontalFlip(p=0.5),
    A.RandomBrightnessContrast(p=0.3),
    A.HueSaturationValue(p=0.3),
    A.RandomResizedCrop(640, 640, scale=(0.5, 1.0)),
], bbox_params=A.BboxParams(format='yolo'))
Test-Time Augmentation (TTA)

例外:推論時也做增強!對同一張圖做多次不同增強,取平均預測結果。犧牲速度換精度,常用於比賽。YOLO 的 TTA 通常做:原圖 + 水平翻轉 + 多尺度(×3 推論時間換 ~1-2% mAP)。

Mosaic 實作 (Pseudo-code)
def mosaic_augment(images, labels, img_size=640):
    """Combine 4 images into one mosaic."""
    s = img_size
    # Random center point
    cx, cy = random.randint(s//4, 3*s//4), random.randint(s//4, 3*s//4)
    mosaic_img = np.zeros((s, s, 3), dtype=np.uint8)

    for i, (img, lbl) in enumerate(zip(images[:4], labels[:4])):
        if i == 0:    # top-left
            mosaic_img[:cy, :cx] = resize(img, (cy, cx))
        elif i == 1:  # top-right
            mosaic_img[:cy, cx:] = resize(img, (cy, s-cx))
        elif i == 2:  # bottom-left
            mosaic_img[cy:, :cx] = resize(img, (s-cy, cx))
        else:         # bottom-right
            mosaic_img[cy:, cx:] = resize(img, (s-cy, s-cx))
        # Adjust bounding box coordinates accordingly
    return mosaic_img, adjusted_labels

小測驗 Quiz

Q1: Mosaic augmentation 最大的好處是?
減少 GPU 記憶體用量
一次讓模型看到更多 context,等效於更大 batch diversity
加快推論速度
Q2: CutMix 跟 Cutout 的最大差別是?
CutMix 用圓形,Cutout 用矩形
CutMix 用在推論,Cutout 用在訓練
CutMix 貼上另一張圖的內容並混合標籤,Cutout 只是遮黑
Q3: YOLOv8 為什麼在最後 10 個 epoch 關閉 Mosaic?
為了讓推論速度更快
讓模型最後階段適應真實圖片分佈,提升最終精度
因為 Mosaic 會導致 overfitting
