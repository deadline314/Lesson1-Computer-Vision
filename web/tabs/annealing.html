模擬退火

Simulated Annealing

模擬退火是一種受冶金學啟發的全域最佳化演算法。 就像金屬加熱再緩慢冷卻,讓原子找到低能量排列一樣,SA 允許搜索過程「偶爾接受更差的解」, 藉此跳出局部最小值,最終找到接近全域最佳的解。

冶金學比喻
鍛造時將金屬加熱到高溫再慢慢冷卻(退火),讓晶體結構達到低能量狀態。SA 把「解」當原子,把「成本函數」當能量。
為什麼需要 SA?
梯度下降只能往下走,容易卡在局部最小值。SA 在高溫時大膽探索,低溫時精細收斂,有機會找到全域最佳解。
溫度的意義
溫度 T 控制「接受壞解的機率」。T 高 → 幾乎什麼都接受(探索)；T 低 → 只接受更好的解(開發)。這就是 exploration vs exploitation。
何時用 SA vs GD?
目標函數不可微、離散、多峰(很多局部最小值)時用 SA。若函數平滑凸形,直接用梯度下降更快。

演算法步驟

1. 初始化 — 設定初始解 x₀、初始溫度 T₀、冷卻率 α
2. 產生鄰居 — 在目前解附近隨機產生新解 x'
3. 計算能量差 — ΔE = f(x') − f(x)
4. 接受判定 — 若 ΔE ≤ 0 → 一律接受；若 ΔE > 0 → 以機率 P = exp(−ΔE/T) 接受
5. 降溫 — T ← T × α（通常 α ∈ [0.90, 0.999]）
6. 重複 — 回到步驟 2,直到 T 足夠低或迭代次數用完
接受機率公式
P(accept) = exp(−ΔE / T)   when ΔE > 0
ΔE = cost_new − cost_current  |  T = 目前溫度  |  T 高 → P ≈ 1  |  T 低 → P ≈ 0
Pseudo-code
def simulated_annealing(f, x0, T0, alpha, max_iter):
    x = x0
    best = x
    T = T0
    for i in range(max_iter):
        x_new = neighbor(x)
        dE = f(x_new) - f(x)
        if dE < 0 or random() < exp(-dE / T):
            x = x_new
        if f(x) < f(best):
            best = x
        T = T * alpha
    return best

實際應用

旅行推銷員問題 (TSP)

尋找經過所有城市一次的最短路線。鄰居解 = 隨機交換兩個城市的順序。這是 NP-hard 問題,SA 可以在合理時間找到近似最佳解。

神經架構搜索 (Neural Architecture Search)

搜索最佳神經網路結構(層數、寬度、連接方式)。目標函數不可微,SA 可用於離散結構空間的探索。

超參數調整 (Hyperparameter Tuning)

學習率、batch size、dropout rate 等超參數的最佳組合。SA 比 grid search 更有效率,比 random search 更有方向性。

特徵選擇 (Feature Selection)

從上百個特徵中選出最佳子集。鄰居解 = 隨機加入或移除一個特徵。SA 可以避免 greedy selection 的局部最佳陷阱。

SA 與 Deep Learning 的關係
在深度學習中,SA 的精神已融入許多技術: Learning rate warmup 就像從高溫開始; cosine annealing scheduler 的名稱直接源自退火; SGD 本身的 noise 也提供了跳出鞍點的能力。 理解 SA 有助於理解為什麼隨機性在優化中如此重要。
